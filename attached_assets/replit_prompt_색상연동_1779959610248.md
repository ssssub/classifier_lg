# Replit Agent 프롬프트 — 색상 → 코드/재질 자동 연동

아래 두 파일을 프로젝트에 추가했어 (`lg_products_sku.json`, `recommender.py`).
이걸 기준으로 결과 카드의 색상 선택 기능을 연동해줘.

## 데이터 구조
`lg_products_sku.json` 은 79개 대표 모델 배열. 각 모델은 이렇게 생겼어:

```json
{
  "rep_code": "B315DS11",
  "name": "LG 일반냉장고",
  "install_type": "프리스탠딩",
  "door_count": "2도어",
  "energy_grade": 1,
  "capacity_total": 317,
  "price_min": 620000,
  "price_max": 700000,
  "colors": ["다크 그라파이트", "퓨어", "화이트"],
  "variants": [
    { "code": "B315DS11", "color": "다크 그라파이트", "material": "네이처(메탈)", "price": 620000 },
    { "code": "B312S31",  "color": "퓨어",          "material": "네이처(메탈)", "price": 663000 },
    { "code": "B315W11",  "color": "화이트",        "material": "네이처(메탈)", "price": 700000 }
  ]
}
```

핵심: 추천은 **대표 모델 단위**로 하고(기존 로직 그대로), 사용자가 결과 카드에서
**색상을 고르면 그 색상의 `code` / `material` / `price` 가 자동으로 바뀌게** 한다.

## 해야 할 일

1. 추천 로직은 **변경하지 마.** 기존처럼 대표 모델 단위로 추천 결과를 만든다.

2. 결과 카드 각각에:
   - 그 모델의 `colors` 배열을 **색상 선택 칩/스와치**로 표시
   - 기본 선택은 `variants[0]` (또는 price_min 색상)
   - 색상을 누르면 → 같은 모델의 `variants`에서 해당 `color`를 찾아
     **표시되는 모델코드(`code`)·도어 재질(`material`)·가격(`price`)을 그 값으로 교체**
   - 추정/디코딩 하지 말고 데이터 그대로 매핑만 한다.

3. 백엔드에 색상 조회 엔드포인트가 필요하면 `recommender.py`의
   `get_variant(rep_code, color)` 를 그대로 써. 코드/재질/가격이 담긴 dict를 반환해.

4. 색상이 1종뿐인 모델은 색상 선택 UI를 숨기거나 비활성화해서 깔끔하게.

## 디자인
기존 LG 톤(레드 포인트, 미니멀) 유지. 색상 칩은 작고 동그란 스와치 + 색상명,
선택된 칩은 테두리 강조. 모델코드/가격이 바뀔 때 부드러운 전환.
