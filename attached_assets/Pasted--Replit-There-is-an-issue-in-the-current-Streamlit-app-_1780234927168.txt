There is an issue in the current Streamlit app where button styles are interfering with each other.

Problem:

* Option card buttons should be left-aligned.
* The “See Results Now →” button should be center-aligned.
* The “← Previous” and “← Start Over” buttons should be small, clean bottom navigation buttons.
* However, the current CSS is being applied globally to `button[data-testid="stBaseButton-secondary"]`, so the styles are bleeding into each other.

Please fix this by clearly separating the buttons into three different types.

1. Regular option card buttons

* Class: `lg-option-btn`
* The card should take `width: 100%`
* The internal content should be left-aligned
* The icon should be on the left, the text should be to the right of the icon, and the arrow should be aligned to the far right of the card
* Keep the existing card design

2. “See Results Now →” button

* Class: `lg-skip-btn`
* Do not apply the option card button style to this button
* Center-align it on the page
* `width: auto`
* `min-width: 220px`
* The internal text should also be center-aligned
* Use a natural margin, around `34px auto 56px auto`, to create balanced spacing above and below

3. “← Previous” and “← Start Over” buttons

* Class: `lg-back-btn`
* Do not apply the option card button style to these buttons
* Remove border and shadow
* Make them look like simple text buttons
* Place “Previous” on the bottom left and “Start Over” on the bottom right
* These buttons must not look like option cards

Important:
Do not directly style all `button[data-testid="stBaseButton-secondary"]` globally in CSS.

Please use exception handling like this:

```css
button[data-testid="stBaseButton-secondary"]:not(.lg-skip-btn):not(.lg-back-btn) {
  /* option card button styles */
}
```

Or, preferably, apply the card button styles only to the `lg-option-btn` class.

Since Streamlit’s internal button structure includes `button > div > p`, please also override the internal `div` and `p` styles separately for each button class.
